# 프로젝트 기획서: Instant Replay GIF Maker

## 1. 프로젝트 개요
사용자가 지정한 단축키를 누르면, 누르기 직전의 과거 1분간의 화면 중 원하는 영역만 선택하여 즉시 GIF로 저장해 주는 백그라운드 유틸리티 프로그램.

---

## 2. 핵심 기능 (Features)
1. **상시 백그라운드 버퍼링:** 시스템 자원(RAM, CPU) 소모를 최소화하며 최근 1분(60초) 분량의 화면을 상시 기억.
2. **글로벌 단축키 감지:** 다른 프로그램을 사용 중일 때도 작동하는 시스템 전역 단축키 지원.
3. **화면 영역 및 시간 선택:** 단축키 입력 즉시 화면이 일시 정지되며, 마우스 드래그로 캡처할 영역을 직관적으로 지정한 후, 저장할 과거 시간(초)을 설정.
4. **고화질 GIF 인코딩:** 선택된 영역과 시간만큼의 프레임을 잘라내어 색상 깨짐 없는 고품질 GIF로 자동 변환 및 저장.

---

## 3. 구성 요소 및 디렉토리 구조 (Components)

프로젝트는 유지보수와 기능 확장을 위해 역할별로 모듈을 분리하여 구성한다.

### 📁 Directory Structure
* **`main`**: 프로그램 진입점 및 전체 프로세스 오케스트레이션
* **`config`**: 사용자 설정 파일 (단축키, 버퍼 시간, 프레임 레이트, 저장 경로 등)
* **`core/` (핵심 모듈)**
  * `recorder`: 화면 캡처 및 메모리 버퍼 관리기
  * `selector`: 영역 선택 UI 매니저
  * `processor`: 영상 컷편집 및 GIF 렌더링 엔진

---

## 4. 핵심 로직 흐름 (Core Logic)

프로그램이 실행되고 종료될 때까지 내부적으로 처리되어야 하는 논리적 흐름.

### Step 1: 초기화 및 대기 (Initialization)
* 설정 파일(`config`)을 읽어와 버퍼 시간(예: 60초)과 FPS(예: 10)를 설정한다.
* 계산된 최대 프레임 수(60초 * 10FPS = 600 프레임)만큼만 저장되는 **순환 큐(Circular Queue)** 자료구조를 메모리에 생성한다.
* 백그라운드 스레드에서 화면을 계속 캡처하여 큐에 밀어 넣는다. (가장 오래된 프레임은 자동으로 삭제됨)

### Step 2: 단축키 인터럽트 (Trigger)
* 사용자가 지정된 단축키를 누르는 순간, 백그라운드 캡처 스레드에서 현재까지 쌓인 **순환 큐의 데이터 복사본(Snapshot)**을 뜬다.
* 화면 캡처 스레드는 멈추지 않고 계속 백그라운드에서 돌아가게 둔다. (GIF 생성 중에도 다음 녹화를 위해)

### Step 3: 영역 지정 및 시간 설정 UI (UI Overlay)
* 방금 뜬 스냅샷의 가장 마지막 프레임(현재 화면)을 모니터 전체 크기의 경계선 없는(Frameless) 반투명 창으로 띄운다.
* 사용자가 마우스를 클릭하고 드래그하여 놓을 때까지의 이벤트 좌표를 추적한다.
* 마우스 드래그가 끝나면, 선택된 영역 근처에 간단한 UI를 띄워 과거 몇 초(최대 버퍼 시간 내) 분량을 GIF로 만들지 입력받는다.
* 시작점 `(x1, y1)`과 끝점 `(x2, y2)`의 좌표 및 사용자가 설정한 시간(초)을 반환하고 UI 창을 닫는다.

### Step 4: 미디어 처리 및 렌더링 (Processing)
* Step 3에서 얻은 좌표값과 시간(초)을 이용해, 확보된 프레임 배열 중 사용자가 지정한 시간만큼의 최신 프레임들만 추출하고 해당 좌표 크기만큼 일괄적으로 자른다(Crop).
* 잘라낸 프레임들을 모아 GIF 포맷으로 인코딩한다. 이때 화질 보존을 위해 팔레트(Palette) 최적화 알고리즘을 적용한다.
* 지정된 `output` 폴더에 현재 날짜와 시간을 파일명으로 하여 GIF를 최종 저장한다.
